# ICF Editor

一个用于修改那个中间是圆形然后旁边有一圈白色塑料按键的游戏以及同系列其他游戏的icf文件的 Windows 图形化工具。

本工具处理的是特定游戏使用的二进制 ICF 文件，与 IAR Embedded Workbench 的 Linker Configuration File（.icf）无关。

程序能够识别 Pack、App、Patch、Opt 以及已安装环境中的内部补丁状态记录，并在保存时自动完成数据序列化、CRC 校验和加密。

界面和文档中的部分术语是作者根据对文件结构的理解进行的命名或翻译，可能与官方称谓存在差异。

## 功能

- 打开和编辑 `icf` 文件
- 新增 Pack、App、Patch 和 Opt 记录
- 兼容安装介质 ICF 与已安装环境 ICF
- 识别普通补丁和内部补丁状态记录
- 自动生成文件名预览
- 自动计算主数据 CRC 和子数据 CRC
- 自动加密并保存 ICF
- 覆盖保存前自动创建 `.bak` 备份
- 支持无须安装 .NET 的 Windows x64 独立版本

## 使用方法

1. 运行 `ICF-Editor.exe`。
2. 点击 **打开**，选择需要编辑的 `icf` 文件。
3. 直接修改表格中的版本、日期和依赖信息。
4. 可通过工具栏新增 Pack、App、Opt 或补丁记录。
5. 检查“生成的文件名预览”是否符合预期。
6. 点击 **保存** 或 **另存为**。

覆盖原文件时，程序会在相同目录创建：

```text
原完整文件名.bak
```

## 文件命名规则

“生成的文件名预览”是根据 ICF 记录字段计算得到的结果。ICF 内部不保存完整文件名。

### Pack 文件

```text
{系统ID}_{系统版本}_{日期时间}_0.pack
```

示例：

```text
ACA_0091.51.01_20240802143251_0.pack
```

### App 完整安装包

```text
{游戏ID}_{应用版本}_{日期时间}_0.app
```

示例：

```text
SDEZ_1.50.00_20250318102642_0.app
```

### App 增量补丁

```text
{游戏ID}_{目标版本}_{目标日期时间}_1_{来源版本}.app
```

示例：

```text
SDEZ_1.51.11_20250824153709_1_1.50.00.app
```

表示从 `1.50.00` 升级到 `1.51.11`。

### Opt 文件

```text
{游戏ID}_{Opt标识}_{日期时间}_0.opt
```

示例：

```text
SDEZ_A005_20250912084327_0.opt
```


## 日期时间规则

文件名中的日期时间统一使用：

```text
yyyyMMddHHmmss
```

例如：

```text
20250824153709
```

表示：

```text
2025-08-24 15:37:09
```

年份使用四位，月、日、时、分、秒分别使用两位，不包含空格、横线或冒号。

## 版本号规则

统一使用：

```text
主版本.次版本.修订版本
```

示例：

```text
1.51.11
91.51.01
```

建议次版本和修订版本固定补足两位：

```text
1.5.1    不推荐
1.05.01  推荐
```

## 补丁版本链规则

补丁中的来源版本和来源日期是独立保存在当前记录中的，不是对上一条记录的动态引用。

例如：

| 版本 | 日期时间 | 来源版本 | 来源日期 |
|---|---|---|---|
| 1.50.00 | 2025-03-18 10:26:42 | - | - |
| 1.51.11 | 2025-08-24 15:37:09 | 1.50.00 | 2025-03-18 10:26:42 |
| 1.52.06 | 2026-02-11 19:08:35 | 1.51.11 | 2025-08-24 15:37:09 |

应满足：

```text
当前补丁的来源版本 = 上一个版本的版本号
当前补丁的来源日期 = 上一个版本的日期时间
```

如果修改了 `1.51.11` 的日期，还应同步修改所有以 `1.51.11` 为来源版本的记录中的“来源日期”。编辑器目前不会自动同步这些独立字段。

## 支持的记录类型

- Pack
- App
- Opt
- Patch
- 已安装环境中的内部补丁状态记录

界面中的类型名称用于区分不同记录，不代表 ICF 文件中保存了相应的文字名称。

## 建议的记录排列顺序

```text
1. Pack
2. App
3. Patch（按照升级顺序）
4. Opt（例如按照 A005、A011、A021 排序）
5. 内部状态记录
```

程序保存时默认保留当前记录顺序，不会自动排序。
